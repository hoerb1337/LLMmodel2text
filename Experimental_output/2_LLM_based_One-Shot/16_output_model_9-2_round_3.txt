The process "Mortgage Offering" involves one main pool as participant. A Pool is the graphical representation of a Participant in a Collaboration. Within this pool, two roles are involved reflected in two different lanes: Customer Service Representative and Administration. A Lane is a sub-partition within a Pool and will extend the entire length of the Pool. Lanes are used to organize and categorize Activities.

The process begins with a start event in the "Customer Service Representative" lane. The Start Event indicates where a particular Process will start. The Customer Service Representative initiates the process by sending out a mortgage offer in the task "Send Out Mortgage Offer". A Task is an atomic Activity that is included within a Process. A Task is used when the work in the Process is not broken down to a finer level of Process detail. Once the offer is sent out, the process moves to an Event-Based Gateway.

An Event-Based Gateway represents a branching point in the process where the alternative paths that follow the Gateway are based on events that occur, rather than the evaluation of expressions using process data (as with an Exclusive or Inclusive Gateway). Three possible events can occur: the customer can respond that they are not interested, the completed offer documents can be received, or two weeks can pass without a response.

If the customer is not interested or two weeks pass without a response, the process moves to a Sub-Process where the case file is updated and archived. A Sub-Process is an Activity whose internal details have been modeled using Activities, Gateways, Events, and Sequence Flows. The process then ends with an End Event.

If the completed offer documents are received, the process moves to the Administration's lane. The Administration then assesses the mortgage details in the "Assess Mortgage Details" task. After assessing the details, the process reaches a decision point, represented by the "Exclusive Gateway". A diverging Exclusive Gateway (Decision) is used to create alternative paths within a Process flow, whereas only one of the paths can be taken.

If all documents are available, the process moves to a Sub-Process where the mortgage application is finalized. The process then ends with an End Event indicating that the mortgage is in the system. If not all documents are available, the process moves to a Task where the outstanding documents are requested. If the documents are received, the process returns to the Exclusive Gateway. If the documents are not received within a certain time period, the process moves to a Sub-Process where the case details are updated and archived. The process then ends with a Cancel End Event. A Cancel End Event is thrown when an Activity is cancelled before normal completion.
